WHAT IS JAVASCRIPT?

  • "Great flexibility comes with errors."
  • A language used for web programming.
  • Serves as a client side script/program which is stored in the server and linked with a webpage.
  • Invented at Netscape.
  • Alternative languages: clash & vbscript

SCIPT PLACEMENTS

  • Linked/External
    - Problem: Before continuing rendering the page, the browser would have to first completely fetch and compile the script. This is important to take note of because a second latency in loading a page will have a big impact, especially to users.
    - Attributes:
    1. defer
      - means to fetch the script after the page has loaded regardless of placement in the script
    2. async
      - means to fetch the script then load the page while waiting for the script to be fetched. Once it has been fetched, continue with parsing the script.
  • Embedded/Internal
    - script that is placed anywhere in the HTML document.

  • Inline
    - usual prefix is 'on'.
    - invoke a method.

  • < noscript >
    - used when a browser does not support scripting or is disabled.

    • Primitive Types
      • Numbers
        - Special Numbers:
         - 1/0 = infinity
         - -1/0 = -infinity
         - 0/0 = NaN
      • Strings
        - can be enclosed in a single quote or double quote.
        - can also use template literals.
      • Booleans
        - 0, "", false, null, undefined & NaN are considered false.
      • Null
      • Undefined
      • Symbols
    • Object Types (Wrapper Types)
      - string, number, boolean and date & time(zero based)
    • Regular Expressions (RegExp Object)
    • Mathematical Functions & Objects (Math Objects)
    • Equality and Identity (Strict Equality)
        1 = = '1'   true
        1 = = = '1'   false
        1 ! = true   false
        1 ! = = true   true
    • Implicit Type Conversions
      - if one variable is a string, the result would automatically be concatenation.
      - null is equivalent to 0 & undefined is equivalent to NaN in arithmetic operations.
      NOTE: use object functions for explicit type conversions
        ex. console.log(8 + Number('9'));   //results to 17
  2. Functions
    - First class objects.
    - If no return is done, the return value would be undefined.
    - 'arguments' object is like an array object that corresponds to the arguments being passed to a function and can only be used inside of that function.
    - Provides the capability to pass functions as arguments on other functions.
    - Nesting functions are also available with the intention of decluttering.
    ex.  function x (){
      function y (){}
      //function y is accessible at this point
     }
     //function y is already inaccessible at this point

  3. Arrays
    - It has dynamic sizing.
    - Allows "sparse" arrays: elements in between which are empty.
    - The length is always one more higher than the actual size.
    - May contain mized types.
      var a1 = newArray(10)  //reserves 10 spaces which are empty
      var a2 = [10]  //specifying an array containing the value 10
    - Some methods available: .push() .pop() .unshift() .shift() .sort() .reverse() .map() .reduce() .filter()

  4. User-defined Objects
    - Ability to create objects on the fly.
    - Makes use of the prototype approach.
    - Uses the associative array syntax [''] & dot syntax for accessing and creating properties.
    - Accessing non-existent properties would return a value of undefined.
    - It is a convention to use Pascal case for the object constructor function.
    - JSON, a popular data interchange format, allows object conversions to and from JSON-formatted strings (.stringify() & .parse())

EVENT HANDLING

  1. Event Propagation
    • Bubbling Phase
      - used to trigger from inner to outer.
      - used by IE.
    • Capturing Phase
      - used to trigger from outer to inner.
    • DOM Order
      Capture -> At Target -> Bubble
  2. Event Handler Registration
    • Inline HTML event attribute
      - This is not recommended but fully supported.
      onclick = 'handler()'
    • pre-DOM L2
      - This is preferred and universally supported but non-standard.
      - Can only register one handler
      - To remove a handler, assign any invalid value.
      button.onclick = handler;
    • DOM L2 event registration
      - Can register more than one event handler.
      .addEventListener(nameOfEvent, handler, boolean);
      *for the 3rd argument: false = bubbling phase & true = capturing phase
      - Other methods: removeEventHandler() & dispatchEvent()
    • Microsoft-specific
      - Available for IEv6 to IEv10.
      - Intention: vendor lock.
      - Methods: .attachEvent() .detachEvent() & .fireEvent()
