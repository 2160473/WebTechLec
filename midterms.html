Cascading Style Sheet


WHAT IS CSS?

  • Labelled as a declarative programming language, Cascading Style Sheets (CSS) are used in formatiing the layout of a webpage wherein rules for HTML documents are described one by one as browsers handle their rendering.

CSS HISROTY

  • written by Marc Andreesen in 1994

CSS DECLARATIONS

  • - a CSS rule-set is divided into two; a selector and a declaration block

    h1 (selector) {color:yellow; font-size:14px;} (declaration)
    property -- value -- property --value

    Selector - points to the HTML element you want to style
    Declaration Block - contains one or more declarations separated by semi-colons
    ** CSS Declarations always end with a semi-colon

    • Shorthand Properties
      - allows authors to specify multiple values
      - one example of a shorthand property is the CSS background propery, that able's to define values of
      - background-color
      - background-image
      - background-repeat
      - background-position

      * another example
      font-style: italic;
      font-weight: bold;
      font-size: .10em;
      line-height: 1.4;
      font-family; Arial, helvetica;
      this can be shortened to: font: talic bold .10em/1.4 Arial, helvetica;

      ** CSS has a universal shorthand property "all"
      -this applies its value to every property in the document
      1. inherit - Sets the property value applied to a selected element to be the same as that of its parent element.
      2. initial - Sets the property value applied to a selected element to be the same as the value set for that element in the browser's default style sheet. If no value is set by the browser's default style sheet and the property is naturally inherited, then the property value is set to inherit instead.
      3. revert - Reverts the property to the value it would have had if the current origin had not applied any styles to it. In other words, the property's value is set to the user stylesheet's value for the property (if one is set), otherwise the property's value is taken from the user-agent's default stylesheet.
      4. unset - Resets the property to its natural value, which means that if the property is naturally inherited it acts like inherit, otherwise it acts like initial.


      example:

      body {
      color: green;
      }

      .inherit a {
      color: inherit;
      }

      .initial a {
      color: initial;
      }

      .unset a {
      color: unset;
      }

      *Vendor Specific Extensions
      - vendor prefixes
      - vendor specific extensions are usd by browser as a prefix for the names of experimental (non-standard) css properties (e.g -webkit-)

    WHAT IS JAVASCRIPT?

    • "Great flexibility comes with errors."
    • A language used for web programming.
    • Serves as a client side script/program which is stored in the server and linked with a webpage.
    • Invented at Netscape.
    • Alternatives to JavaScript : flash , applets , and activeX
    • Alternative languages: clash & vbscript


    HOW? < script type='text/javascipt' src='path/to/script.js' %gt; < /script >
    • The script may be placed anywhere
    • It may implemented in two types: top-level (outside of a function, executed as the scrip is loaded) and inline (located inside a function, and is only executed when it is called).
    • The functionality of scripts are used upon interaction.
    • Normally, before the rendering of a page continues, scripts are fetched, compiled and executed.
    • Mobile users tend to be more impatient due to slow connectivity issues. When the first page of a website slowly loads, they tend to go to another site. Because scripts also contributes to the weight of the site, one must consider the placement and size of the script for it is one factor that affects user satisfaction.

    SCIPT PLACEMENTS

    • Linked/External
      - Problem: Before continuing rendering the page, the browser would have to first completely fetch and compile the script. This is important to take note of because a second latency in loading a page will have a big impact, especially to users.
      - Attributes:
      1. defer
        - means to fetch the script after the page has loaded regardless of placement in the script
      2. async
        - means to fetch the script then load the page while waiting for the script to be fetched. Once it has been fetched, continue with parsing the script.

      Note: some browsers don't have support for these attributes. As an alternative solution to hte problem, scripts may be placed at the end of the page.

    • Embedded/Internal
      - script that is placed anywhere in the HTML document, particularry in between < script > < /script > ./li>
    • Inline
      - invoke a method.
      - values are placed inside html elements.
      - These are usually event attributes and are triggered when users interact with the document.
      - usual prefix is 'on' (e.g. onclick).
    • < noscript >
      - provides alternate content and is used when a browser does not support scripting or is disabled.

        - used to trigger from outer to inner.

        - This is not recommended but fully supported.
        onclick = 'handler()'
