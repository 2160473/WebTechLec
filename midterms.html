WHAT IS JAVASCRIPT?

  • "Great flexibility comes with errors."
  • A language used for web programming.
  • Serves as a client side script/program which is stored in the server and linked with a webpage.
  • Invented at Netscape.
  • Alternatives to JavaScript : flash , applets , and activeX
  • Alternative languages: clash & vbscript


HOW? < script type='text/javascipt' src='path/to/script.js' %gt; < /script >
  • The script may be placed anywhere
  • It may implemented in two types: top-level (outside of a function, executed as the scrip is loaded) and inline (located inside a function, and is only executed when it is called).
  • The functionality of scripts are used upon interaction.
  • Normally, before the rendering of a page continues, scripts are fetched, compiled and executed.
  • Mobile users tend to be more impatient due to slow connectivity issues. When the first page of a website slowly loads, they tend to go to another site. Because scripts also contributes to the weight of the site, one must consider the placement and size of the script for it is one factor that affects user satisfaction.

SCIPT PLACEMENTS

  • Linked/External
    - Problem: Before continuing rendering the page, the browser would have to first completely fetch and compile the script. This is important to take note of because a second latency in loading a page will have a big impact, especially to users.
    - Attributes:
    1. defer
      - means to fetch the script after the page has loaded regardless of placement in the script
    2. async
      - means to fetch the script then load the page while waiting for the script to be fetched. Once it has been fetched, continue with parsing the script.

    Note: some browsers don't have support for these attributes. As an alternative solution to hte problem, scripts may be placed at the end of the page.

  • Embedded/Internal
    - script that is placed anywhere in the HTML document, particularry in between < script > < /script > ./li>
  • Inline
    - invoke a method.
    - values are placed inside html elements.
    - These are usually event attributes and are triggered when users interact with the document.
    - usual prefix is 'on' (e.g. onclick).
  • < noscript >
    - provides alternate content and is used when a browser does not support scripting or is disabled.

JAVASCRIPT EXECUTION ENVIRONMENT


An avenue wherein all codes are combined and associated within a single page.
  1. Core JavaScript( EcmaScript ) Language and API
  2. Browser Object Model API (non-standardized)
    - using the console of the browser.
    - Window (global), Navigator(the browser itself), Screen(current screen), Location(URL), History
  3. Document Object Model API
    • Most important API for client-side scripting.
    • access to HTML document within scripts associated with the webpage.
    • It is managed by W3C.
    • Nodes
      - primary data type of DOM.
      - represents HTML document components<
      ex. document node, document type node, element node & text node
      note: white spaces are considered as text nodes.
    • Functionalities
      - Retrieve references to nodes.
      - Traverse the DOM tree.
      - Construct or copy nodes.
      - Manipulate the DOM tree.
  4. Miscellaneous
    - Geolocation , IndexedDB , Local Storage , Push Notifications , Service Workers , Web Sockets , Web Workers , XMLHttpRequest(AJAX)
